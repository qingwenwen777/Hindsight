## 3. 核心概念

这个系统和普通看盘软件最大的不同：**它强迫你记录"为什么买卖"，并在事后帮你复盘，对抗投资中的认知偏差。**

- **交易（Transaction）**：一笔真实的买入/卖出记录。
- **决策日志（Journal）**：每笔交易**必须**附带的"投资逻辑"，写明为什么做这个决定、目标价、止损、风险。提交后会锁定，事后不能偷偷改，保证复盘的真实性。
- **复盘（Review）**：过一段时间回看当初的决策，判断逻辑是否成立、是运气还是实力。
- **关注列表（Watchlist）**：只想跟踪、还没下手的标的。
- **冷静期**：提交交易前有 30 秒倒计时，检测到疑似"复仇交易"（刚亏完马上反手）会延长，逼你冷静。

## 4. 功能导航（左侧菜单）

### 概览
- **仪表盘**：账户总览，持仓市值、盈亏、关键指标一屏看全。
- **持仓**：当前持有的股票明细，成本、市值、浮动盈亏。
- **现金流**：现金账户的入金/出金/分红流水。
- **关注**：跟踪但未持有的标的，可一键搜索添加。

### 交易
- **交易记录**：所有买卖历史，可筛选查看。
- **决策日志**：所有决策的逻辑记录，是复盘的核心资产。

### 分析
- **收益分析**：时间加权/资金加权收益率等。
- **基准对比**：你的组合 vs 大盘指数，看是否跑赢。
- **暴露分析**：行业/市场/个股的集中度，提示是否过度集中。
- **情绪审计**：把交易时记录的情绪和最终盈亏对照，看情绪化交易的代价。
- **报表中心**：导出各类统计报表。

### AI 洞察
- **AI 日报**：系统每天定时为你关注的市场生成一份日报（Markdown 排版，可下载）。日报和"你自己"绑定——盯你的持仓/关注异动、触及目标价/止损价、对照当初的决策逻辑、汇总到期复盘和集中度告警，而不是泛泛复述大盘。可在「日报设置」里配置：盯哪些市场、几点生成、异动阈值、语气、语言，以及"重点关注"（行业/主题/关键词）和自定义约束（如"每只不超过 3 句""不碰加密货币"）。也能随时点「立即生成」。文档保留最近 90 天。
- **选股筛选**：用你自己的硬性规则筛股票（如 PE<20 且营收增速>15% 且在关注列表）。筛选是**纯机械、可解释**的确定性计算，不是 AI 预测。筛完可选择「请 AI 点评」——AI 给每只标的多空两方观点、风险、待补充调研方向，**明确不给买卖结论、不预测价格**，点评结果存进 AI 日报列表。

### AI
- **AI 对话**：内置 AI 投资教练。可以问个股看法、让它帮你挑战投资逻辑、复盘某笔交易的得失。它不会替你做决定，而是帮你想得更清楚。

### 设置
- **设置**：基础货币、显示偏好、界面语言（中/日/英）等。

> **顶栏铃铛**：聚合两类提醒——复盘到期提醒 + 价格提醒（持仓/关注标的触及你在决策日志里设的目标价/止损价时即时提示）。点开可跳转处理、标记已读。

## 5. 典型操作流程

### 5.1 添加一只新股票
**方式 A（推荐）**：Ctrl+K 搜索 → 库里没有时点「从数据源发现」的结果 → 自动添加并同步行情。

**方式 B**：进入「关注」页 → 搜索框输入名称/代码 → 本地没有会列出数据源候选 → 点「添加并同步」。

添加后历史行情在后台拉取，通常几秒到十几秒就绪，刷新个股页即可看到 K 线。

### 5.2 记录一笔交易
1. Ctrl+K → 「新建交易」，或左侧任意位置进入交易录入
2. **第一步 交易信息**：选股票（搜不到可现场添加）、买/卖、数量、价格、日期
3. **第二步 决策日志**（必填）：
   - 决策类型、论点类别、预期持有期
   - 目标价、止损价、退出条件
   - 信心评分、当前情绪
   - **投资逻辑（≥100 字）**：为什么做这笔交易
   - 主要风险
4. 通过 30 秒冷静期后提交。手续费由后端按市场规则自动计算。

> 提交后日志会**锁定**，不能再改，这是刻意设计——保证将来复盘时看到的是你当时的真实想法。

### 5.3 用 AI 复盘
进入「AI 对话」，可以这样问：
- "帮我复盘一下我买入特斯拉那笔交易，当时的逻辑现在还成立吗？"
- "我最近是不是有追涨杀跌的倾向？"
- "茅台现在的估值贵吗？帮我列出多空两方观点。"

## 8. 常见问题

**Q：搜索某只股票什么都查不到？**
A：先确认输入的是名称或代码。若库里确实没有，应该会出现「从数据源发现」的候选，点一下添加即可。如果连发现结果都没有，可能是该标的不在支持的四个市场内，或数据源暂时不可用，稍后再试。

**Q：添加后个股页没有 K 线？**
A：历史行情在后台拉取，稍等几秒刷新页面。若一直没有，可能是该股代码在数据源里对不上，可换搜索词重试。

**Q：A股数据准吗？**
A：A股走新浪/腾讯数据源（东方财富在服务器所在地被屏蔽，已做替代），数据完整含成交量，更新可能比盘后稍晚，正常使用没问题。

**Q：决策日志写错了能改吗？**
A：提交后会锁定，不能修改——这是为了保证复盘真实。可以新建一篇日志或在复盘里补充说明。

**Q：网站打不开？**
A：确认地址带端口 `:3100`、用的是 `http`。部分公司/校园网络会拦截非标准端口，换网络试试。

## 9. 运维速查（给管理员）

> 普通使用者可忽略本节。

**服务器**：`ssh root@154.36.185.85`，项目目录 `/opt/tradeai`

**容器**：
```bash
docker ps --filter name=tradeai           # 查看状态
docker logs tradeai-backend --tail 50     # 后端日志
docker restart tradeai-backend            # 重启后端
```

**重新部署（拉取代码改动后）**：
```bash
cd /opt/tradeai
docker compose -f docker-compose.prod.yml --env-file .env.prod up -d --build
```

**预置/补充股票**（在后端容器内执行）：
```bash
docker exec -w /app tradeai-backend python -m scripts.seed_stocks            # 全部市场
docker exec -w /app tradeai-backend python -m scripts.seed_stocks --market CN # 仅A股
docker exec -w /app tradeai-backend python -m scripts.seed_stocks --list      # 只看清单
```

**扩充股票池（成分股，让选股筛选有广度）**：
```bash
docker exec -w /app tradeai-backend python -m scripts.seed_universe            # 全部市场
docker exec -w /app tradeai-backend python -m scripts.seed_universe --market US # 仅美股
# 或经 API 后台触发 + 查完备度：
curl -X POST 'http://localhost:8000/api/v1/admin/seed-universe?market=US'
curl 'http://localhost:8000/api/v1/admin/universe-status'
```

**AI 洞察相关**：
```bash
# 手动生成某市场日报（前端「AI 日报」页也有「立即生成」按钮）
curl -X POST 'http://localhost:8000/api/v1/insights/daily/generate?market=US'
# 同步财务（选股筛选依赖 PE/ROE/营收增速等财务字段）
curl -X POST 'http://localhost:8000/api/v1/admin/sync/financials?market=US'
```

> 选股筛选需要财务数据：新登记的股票要先 `sync/financials` 才能按 PE/营收增速等条件筛到。日报定时生成由 `ENABLE_SCHEDULER=true` + 「日报设置」里的时间驱动。

**手动触发行情同步**（API）：
```bash
curl -X POST 'http://localhost:8000/api/v1/admin/sync/prices?market=US'  # 按市场
curl 'http://localhost:8000/api/v1/admin/sync/logs?limit=20'             # 查同步日志
```

**关键配置**（`/opt/tradeai/.env.prod`）：
- `ENABLE_SCHEDULER=true`：定时同步开关
- `ANTHROPIC_API_KEY` / `AI_BASE_URL` / `AI_MODEL`：AI 服务配置
- `PUBLIC_API_URL` / `EXTRA_CORS_ORIGINS`：前端访问后端的公网地址与跨域白名单

**数据**：SQLite 库在 `/opt/tradeai/data/stock.db`，备份脚本 `docker exec -w /app tradeai-backend python -m scripts.backup`
